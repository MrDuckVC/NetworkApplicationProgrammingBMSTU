## 3. Теоретическая база применения JavaScript

При разработке бизнес-логики приложения были применены следующие современные стандарты и архитектурные концепции (ES6+):

1. **Модель обработки событий (Event-Driven Programming):**
   Отказ от устаревшего паттерна inline-обработчиков (атрибуты `onclick` в HTML) в пользу метода `addEventListener`. Инициализация приложения происходит строго по событию `DOMContentLoaded`, что предотвращает блокировку рендеринга (Render Blocking) интерфейса браузером.
2. **Архитектурный паттерн «Стратегия» (Strategy):**
   Вместо использования громоздких и трудно масштабируемых конструкций `switch-case` для выбора математической операции, применена структура данных «Словарь» (мапы объектов). Операции строго разделены по арности на унарные (мгновенного действия) и бинарные (отложенного действия). Это реализует принцип Open-Closed Principle (SOLID).
3. **Безопасная работа с DOM (DOM Manipulation):**
   Изменение визуального состояния элементов производится не через прямую манипуляцию `style`, а через API `classList` (добавление/удаление классов состояний, таких как `.show` для ошибок и `.accent-theme`). Это сохраняет принцип изоляции ответственности (Separation of Concerns) между JS и CSS.
4. **Обработка исключений (Error Handling):**
   Математические ошибки (например, корень из отрицательного числа) перехватываются с помощью конструкции `try...catch`, переводя приложение в безопасное состояние `ifError = true`, блокирующее дальнейший некорректный ввод до принудительного сброса.

## 4. Архитектурные решения и фрагменты кода

### 4.1. Паттерн «Стратегия» для математических операций

Математические функции инкапсулированы в объекты. Это позволило легко внедрить индивидуальную операцию `btn_op_d2s` (конвертация дней в солы) без изменения основной логики контроллера.

```javascript
// Разделение операций по арности (количеству операндов)
const binaryManeuvers = {
    'btn_op_plus': (x, y) => x + y,
    'btn_op_minus': (x, y) => x - y,
    'btn_op_mult': (x, y) => x * y,
    'btn_op_div': (x, y) => x / y,
    'btn_op_percent': (x, y) => x / 100 * y,
};

const unaryManeuvers = {
    'btn_op_sign': x => x * -1,
    'btn_op_sqrt': x => Math.sqrt(x),
    'btn_op_x2': x => x * x,
    'btn_op_factorial': x => factorial(x),
    // Индивидуальная операция предметной области SpaceX
    'btn_op_d2s': x => x / 1.02749
};

```

### 4.2. Обработка событий и перехват исключений

Для выполнения унарных операций используется единый контроллер. Внедрен блок `try...catch` для защиты от критических сбоев движка V8 при выполнении недопустимых математических операций (например, в функции `factorial`, которая генерирует `throw new Error`).

```javascript
function onUnaryButtonClicked(event) {
    if (ifError) return; // Блокировка пульта при наличии ошибки

    try {
        if (selectedOperation) {
            if (b === '') return;
            // Динамический вызов функции из словаря по ID нажатой кнопки
            let rawResult = unaryManeuvers[event.currentTarget.id](+b);
            b = proccessResultValue(rawResult).toString();
            displayNumber(b);
        } else {
            // ... аналогично для первого операнда ...
        }
    } catch (e) {
        ifError = true; // Перевод пульта в режим ошибки
        displayNumber(''); // Вызовет отображение красного блока .error
    }
}

```

### 4.3. Модульность и манипуляция состоянием UI (Темы)

Логика смены цветовых схем вынесена в отдельный модуль `change_theme.js`. Смена темы реализована через изменение атрибута `data-theme` корневого тега `<body>`, на который реагируют CSS-селекторы, меняя фоновое изображение.

```javascript
// change_theme.js
document.addEventListener('DOMContentLoaded', () => {
    // Обработчик выпадающего списка
    document.getElementById('select_theme').addEventListener('change', (event) => {
        const selectedTheme = event.target.value;
        document.body.setAttribute('data-theme', selectedTheme);
    });

    // Обработчик кнопки циклической смены
    document.getElementById('btn_change_theme').addEventListener('click', () => {
        const currentTheme = document.body.getAttribute('data-theme');
        const newTheme = currentTheme === 'dark' ? 'light' : 'dark';

        document.body.setAttribute('data-theme', newTheme);
        document.getElementById('select_theme').value = newTheme; // Синхронизация селекта
    });
});

```

### 4.4. Регистр независимой памяти

Реализован независимый накопитель (`memoryRegister`), позволяющий сохранять промежуточные расчеты перелетов.

```javascript
let memoryRegister = 0; // Инициализация регистра памяти

document.getElementById('btn_mem_plus').addEventListener('click', () => {
    if (ifError) return;
    // getCurrentScreenValue() возвращает текущее введенное число как Number
    memoryRegister += getCurrentScreenValue();
});

document.getElementById('btn_mem_recall').addEventListener('click', () => {
    if (ifError) return;
    let memString = memoryRegister.toString();

    // Подстановка значения из памяти в активный операнд
    if (!selectedOperation) {
        a = memString;
        displayNumber(a);
    } else {
        b = memString;
        displayNumber(b);
    }
});

```
