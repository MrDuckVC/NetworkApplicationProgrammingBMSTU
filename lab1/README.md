## 3. Теоретическая база применения HTML и CSS

При разработке интерфейса были применены следующие ключевые технологии и концепции:

1. **Семантическая разметка (Semantic HTML5):** Отказ от использования исключительно тегов `<div>` в пользу семантически значимых тегов (`<header>`, `<section>`, `<article>`, `<footer>`, `<output>`). Это улучшает доступность приложения (Accessibility) для screen-reader'ов и позволяет поисковым роботам лучше понимать структуру документа.
2. **Модульность (Separation of Concerns):** Строгое разделение структуры (HTML) и представления (CSS). В коде отсутствуют inline-стили (`style="..."`).
3. **CSS Custom Properties (Переменные):**
   Объявление цветов и других параметров в псевдоклассе `:root`. Это закладывает архитектурный фундамент для реализации паттерна смены визуальных тем (Theme Switching) средствами JavaScript в будущих лабораторных работах.
4. **Современные модели компоновки (CSS Grid & Flexbox):**
   * **Flexbox (`display: flex`):** Использован для построения одномерных структур — навигационной панели (Navigation bar) и центрирования основного контента на экране.
   * **CSS Grid (`display: grid`):** Использован для построения двумерной сетки калькулятора, что позволило отказаться от устаревшей блочной верстки (вложенных `<div>` для каждой строки) и сделать код более чистым.

## 4. Архитектурные решения и фрагменты кода

### 4.1. Семантическая структура страницы

Страницы проекта разделены на четкие логические зоны. Для вывода результата вычислений использован специализированный тег `<output>`, а не стандартный `<div>`, что семантически верно описывает назначение элемента.

```html
<section>
    <article>
        <h1>Калькулятор</h1>
        <div class="calc-grid">
            <output id="result" class="result">0</output>

            <button id="btn_op_clear" class="calc-btn secondary">C</button>
            <button id="btn_op_sign" class="calc-btn secondary">±</button>
            </div>
    </article>
</section>

```

### 4.2. Построение сетки с помощью CSS Grid

Для раскладки кнопок калькулятора применена технология CSS Grid. Это позволяет задать количество колонок и отступы без использования лишних узлов в DOM-дереве. Кнопка `=` расширена на две колонки с помощью свойства `grid-column`.

```css
/* Фрагмент файла style.css */
.calc-grid {
    display: grid;
    /* Создаем 4 равные колонки по 50 пикселей */
    grid-template-columns: repeat(4, 50px);
    gap: 5px; /* Интервал между кнопками */
    width: max-content;
}

.calc-btn.execute {
    /* Растягиваем кнопку вычисления на две ячейки сетки */
    grid-column: span 2;
    border-radius: 4px;
}

.result {
    /* Экран калькулятора занимает весь верхний ряд (от первой до последней линии) */
    grid-column: 1 / -1;
    width: 100%;
}

```

### 4.3. Использование CSS переменных и БЭМ-подобного нейминга

Все константы дизайна вынесены в `:root`. Кнопки стилизованы с использованием базового класса `.calc-btn` и классов-модификаторов (`.primary`, `.secondary`), что обеспечивает высокую переиспользуемость кода (DRY).

```css
/* Объявление цветовой схемы миссии */
:root {
    --mars-beige: #d7a384;
    --mars-brown: #b57756;
    --mars-grey: #503530;
    --mars-black: #2e242b;
    --black-50: rgba(0, 0, 0, 0.5);
    /* ... */
}

/* Базовый стиль кнопки панели управления */
.calc-btn {
    background-color: var(--black-50);
    color: var(--white);
    border-color: var(--white-35);
    /* ... */
}

/* Модификатор для основных маневров (математических операций) */
.calc-btn.primary {
    background-color: var(--mars-black);
}

/* Эффект взаимодействия пользователя с пультом */
.calc-btn:hover {
    background-color: var(--white);
    color: var(--black);
}

```
